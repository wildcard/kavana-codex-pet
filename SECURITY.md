# Security Policy

## Supported Versions

| Version | Supported |
| --- | --- |
| `1.x` | Yes |

Open a public GitHub issue for non-sensitive install, corruption, checksum, metadata, or documentation problems.

Do not publish credentials, private photos, raw references, prompt logs, Codex state, backups, or personal data. For sensitive reports, use GitHub’s private vulnerability reporting if enabled.

The release package should contain only curated pet assets, public previews and sources, sanitized metadata, documentation, and local install/verification tools that require no secrets.
